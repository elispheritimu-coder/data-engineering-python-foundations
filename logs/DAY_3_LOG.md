# Day 3 – Python Decision Making & Input Handling

## What I Worked On
- Ran Python scripts from the terminal
- Used `input()` to accept user input
- Used `if / else` statements for decision making
- Compared values using boolean expressions
- Handled invalid input using `try / except`
- Learned the difference between user input and program output

## Key Concepts Learned
- Boolean comparisons return True or False
- `input()` always returns a string
- `int()` converts text to a number
- SyntaxError vs TypeError vs ValueError
- `try / except` prevents program crashes
- Terminal echoes user input; Python only prints what is in `print()`

## Challenges Faced
- Confusion between typed input and printed output
- Understanding where user input actually happens
- Differentiating between runtime behavior and source code

## How I Resolved Them
- Carefully observed terminal behavior
- Clarified that values are entered at runtime, not inside the script
- Used error messages to diagnose issues instead of guessing

- 7/10

## Notes
- Understanding input flow is critical
- Error handling is a core engineering skill
- Feeling confused was part of the learning curve
